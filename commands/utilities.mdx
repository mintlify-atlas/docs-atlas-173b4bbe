## Wiki

Search Wikipedia with comprehensive section navigation.

```
/wiki query:<search term>
```

<ParamField path="query" type="string" required>
  The term to search for
</ParamField>

### Features

- **Smart Search**: Disambiguation menu for multiple results
- **Section Navigation**: Dropdown menu for all article sections
- **Image Support**: Section-specific images from REST API
- **Pagination**: Previous/Next buttons
- **Fallback System**: Classic API if REST fails
- **Session Recovery**: Reloads article if cache expires
- **Rich Formatting**: Clean HTML-to-text conversion

### Navigation

1. Search returns multiple results → Select from dropdown
2. Article loads with Introduction
3. Use section dropdown to jump to specific sections
4. Use Previous/Next for sequential reading
5. Click "Read Article" for full Wikipedia page

## Steal

Clone emojis from other servers with bulk support.

```
/steal emoji:<emoji or URL> [rename:<new name>]
```

<ParamField path="emoji" type="string" required>
  Paste emoji(s) or direct image URL. Supports multiple emojis at once.
</ParamField>

<ParamField path="rename" type="string">
  New name(s). For bulk, separate with commas.
</ParamField>

### Features

- **Bulk Support**: Paste multiple emojis at once
- **URL Support**: Direct image links
- **Batch Renaming**: Comma-separated names
- **Animated Support**: Detects and preserves GIF format
- **Session Pagination**: Navigate through bulk results
- **Success Tracking**: Shows which emojis succeeded/failed
- **Error Handling**: Detailed failure reasons

### Examples

```
/steal emoji::customemoji:
/steal emoji::emoji1: :emoji2: :emoji3: rename:new1, new2, new3
/steal emoji:https://example.com/image.png rename:custom_name
```

<Warning>
Requires **Manage Emojis** permission. Subject to Discord's emoji slot limits.
</Warning>

## Custom

Manage your personal custom role with color, name, and icon.

### Create Custom Role

```
/custom create name:<name> hex:<color> [icon:<attachment>]
```

<ParamField path="name" type="string" required>
  Role name
</ParamField>

<ParamField path="hex" type="string" required>
  Color hex code (e.g., "FF0055")
</ParamField>

<ParamField path="icon" type="attachment">
  Role icon (requires Server Level 2)
</ParamField>

**Features**:
- One role per user
- Automatic positioning below bot's highest role
- Overwrites existing custom role (with confirmation)
- Removes palette color roles automatically
- Soft cap: 100 custom roles (admins bypass)
- Hard cap: 250 total

### Edit Custom Role

```
/custom edit [name:<new name>] [hex:<new color>] [icon:<new icon>]
```

All parameters optional. Updates only specified fields.

### Prune Custom Roles (Admin)

```
/custom prune [execute:<true/false>]
```

<ParamField path="execute" type="boolean" default="false">
  Actually perform deletion (false = dry run)
</ParamField>

**Cleans up**:
- Orphaned roles (user left server)
- Desync entries (role manually deleted)
- Shows statistics before deletion

<Warning>
Custom roles have a soft cap of 100 per server. After reaching this limit, only administrators can create new custom roles. All users can still edit their existing roles.
</Warning>
